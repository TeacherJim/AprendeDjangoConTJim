Tutorial 01

Instala VS Code y Python

Antes de tocar Django necesitas dos herramientas que funcionen de verdad: el lenguaje Python y un editor donde escribir código. En esta misión instalamos ambos, verificamos cada paso y ejecutamos tu primer script.

Inicial ~45 min 10 pasos

Resultado final

Al terminar este tutorial tendrás:

  • Python instalado y accesible desde la terminal con python --version (o equivalente en tu sistema).
  • Visual Studio Code instalado con la extensión oficial de Python de Microsoft.
  • Una carpeta de trabajo llamada django-lab con un archivo hola.py.
  • Un script que pide tu nombre y muestra un saludo en pantalla al ejecutarlo.

Concepto clave

Instalar no significa solo descargar. Significa que tu computadora reconoce el comando cuando lo escribes en la terminal. Si no puedes verificar la versión, todavía no está listo.

Antes de empezar

Revisa esta lista antes de avanzar. Si algo falla, no sigas al siguiente paso hasta resolverlo.

  • Tienes conexión a internet para descargar Python y VS Code.
  • Puedes abrir una terminal: PowerShell, CMD o Terminal según tu sistema operativo.
  • Tienes permisos para instalar programas en tu computadora (o un adulto que te ayude si es necesario).
  • Sabes en qué carpeta guardarás tus proyectos (por ejemplo, Documentos o el Escritorio).
  • Entiendes que copiar comandos sin leerlos es la forma más rápida de perder el hilo.

Nota de Teacher Jim

No te apures. La mayoría de problemas en los tutoriales siguientes vienen de saltarse una verificación aquí. Cada comando que ejecutes debe tener un resultado que puedas comprobar con tus propios ojos.

Mapa de la misión

Diez pasos en orden. Marca cada uno cuando lo completes para guardar tu progreso en este navegador.

  1. Comprender las tres piezas: Python, VS Code y la extensión.
  2. Comprobar si Python ya está instalado.
  3. Instalar Python desde el sitio oficial (con PATH en Windows).
  4. Verificar la instalación de nuevo.
  5. Instalar Visual Studio Code.
  6. Instalar la extensión Python de Microsoft.
  7. Crear la carpeta django-lab.
  8. Escribir el archivo hola.py.
  9. Ejecutar el script desde la terminal.
  10. Checkpoint final: confirmar que todo funciona.
1

Comprender las tres piezas

Qué vamos a hacer

Identificar las tres piezas que necesitas antes de programar con Django: el intérprete de Python, el editor Visual Studio Code y la extensión de Python para VS Code.

Por qué

Muchos estudiantes instalan cosas sin saber para qué sirven. Cuando algo falla, no saben si el problema es Python, el editor o la configuración. Separar las piezas te da un mapa mental claro.

Python ejecuta tu código VS Code donde escribes el código Extensión Python ayuda al editor a entender Python

Las tres piezas explicadas

Python es el lenguaje. Sin él, ningún comando python funcionará en tu terminal.

VS Code es tu taller: archivos, carpetas, terminal integrada y resaltado de sintaxis.

La extensión Python conecta el editor con el intérprete: autocompletado, detección de errores y ejecución con un clic.

Qué debería ocurrir

Deberías poder explicar con tus palabras qué hace cada pieza. No necesitas instalar nada todavía; solo entender el panorama.

Verifica

Comprueba

Responde mentalmente: ¿qué pieza ejecuta el código? ¿cuál lo escribe? ¿cuál ayuda al editor? Si las tres respuestas son distintas, estás listo para el paso 2.

2

Comprobar si Python ya está instalado

Qué vamos a hacer

Abrir una terminal y ejecutar comandos para ver si Python ya está en tu sistema y qué versión tiene.

Por qué

Algunas computadoras ya traen Python instalado. Otros tienen una versión antigua. Antes de descargar nada, comprobamos qué hay.

Código

PowerShell

Prueba en este orden:

python --version
py --version
python3 --version
CMD 
python --version
py --version
Terminal 
python3 --version
python --version

Dónde va

Estos comandos se ejecutan en la terminal de tu sistema operativo, no dentro de VS Code todavía. En Windows: busca "PowerShell" o "Símbolo del sistema" en el menú Inicio.

Qué debería ocurrir

Si Python está instalado correctamente, verás algo como:

Salida esperada 
Python 3.12.4

El número exacto puede variar. Lo importante es que muestre Python 3.x y no un error.

Verifica

Comprueba

Si obtuviste una versión 3.10 o superior, puedes saltar al paso 5 (instalar VS Code). Si viste un error o una versión 2.x, continúa al paso 3.

Error común

"python" no se reconoce como comando — Python no está instalado o no está en el PATH. No es un fallo tuyo; simplemente necesitas instalarlo en el paso 3.

Aparece Python 2.7 — Esa versión es antigua y no sirve para Django 6. Instala Python 3 desde python.org.

3

Instalar Python desde el sitio oficial

Qué vamos a hacer

Descargar e instalar Python 3 desde python.org/downloads. En Windows activaremos la opción que agrega Python al PATH.

Por qué

El instalador oficial es la fuente confiable. Activar PATH en Windows permite que cualquier terminal encuentre el comando python sin configuración extra.

Windows: opción crítica

En la primera pantalla del instalador, marca la casilla "Add python.exe to PATH" (o "Agregar Python al PATH") antes de pulsar "Install Now". Si no la marcas, el paso 4 fallará y tendrás que reinstalar.

Dónde va

La instalación es a nivel de sistema operativo. No necesitas crear carpetas todavía. Acepta las opciones por defecto salvo la casilla PATH en Windows.

Qué debería ocurrir

  • El instalador descarga y copia los archivos de Python.
  • Al finalizar muestra "Setup was successful" o equivalente.
  • En macOS/Linux puede pedirte tu contraseña de administrador.

Verifica

Comprueba

Cierra cualquier terminal que tuvieras abierta antes de instalar. Las terminales antiguas no ven cambios en PATH hasta que las reinicies.

Error común

Instalé Python pero el comando sigue sin funcionar — Casi siempre es porque no marcaste "Add to PATH" en Windows. Desinstala Python desde Configuración > Aplicaciones y vuelve a instalar marcando esa casilla.

Descargué Python desde la Microsoft Store y hay conflictos — Prefiere el instalador de python.org. Es más predecible para aprender.

4

Verificar Python de nuevo

Qué vamos a hacer

Repetir los comandos de verificación en una terminal nueva para confirmar que la instalación funcionó.

Por qué

Instalar sin verificar es adivinar. Este paso convierte "creo que funciona" en "sé que funciona".

Código

terminal 
python --version

En macOS/Linux, si python no responde, prueba python3 --version. En Windows también puedes usar py --version.

Dónde va

Terminal nueva del sistema. Si ya tenías una abierta, ciérrala y abre otra.

Qué debería ocurrir

Deberías ver una línea con Python 3.10 o superior. Anota el número exacto; lo usarás más adelante.

Verifica

Comprueba

Ejecuta también un mini test interactivo:

terminal 
python -c "print('Python responde correctamente')"

La salida debe ser exactamente: Python responde correctamente

Error común

Sigue sin reconocer el comando — Reinicia la computadora. Si persiste, reinstala Python marcando "Add to PATH". No avances al paso 5 hasta resolver esto.

5

Instalar Visual Studio Code

Qué vamos a hacer

Descargar e instalar VS Code desde code.visualstudio.com.

Por qué

VS Code es gratuito, ligero y tiene terminal integrada. Podrías escribir Python en el Bloc de notas, pero perderías resaltado de sintaxis, detección de errores y comodidad al trabajar con carpetas.

Dónde va

Instalación a nivel de sistema, igual que Python. En Windows acepta las opciones por defecto; puedes marcar "Agregar al menú contextual" si quieres abrir carpetas con clic derecho.

Qué debería ocurrir

  • VS Code se abre después de instalar (o desde el menú Inicio / Aplicaciones).
  • Ves la pantalla de bienvenida con opciones como "Open Folder" o "Abrir carpeta".

Verifica

Comprueba

Abre VS Code. Ve a Terminal > Nueva terminal (o Ctrl+`). Debería aparecer un panel de terminal en la parte inferior. Escribe python --version ahí y confirma que muestra la misma versión que en el paso 4.

Por qué importa la terminal integrada

Trabajarás siempre desde la carpeta del proyecto. La terminal de VS Code ya está ubicada en la carpeta que abras, lo que reduce errores de "archivo no encontrado".

6

Instalar la extensión Python de Microsoft

Qué vamos a hacer

Instalar la extensión oficial "Python" publicada por Microsoft dentro de VS Code.

Por qué

VS Code no entiende Python por defecto. La extensión añade IntelliSense, formateo, depuración y la capacidad de seleccionar qué intérprete usar.

Dónde va

  1. En VS Code, haz clic en el icono de Extensiones en la barra lateral (cuatro cuadrados) o pulsa Ctrl+Shift+X.
  2. Busca Python.
  3. Selecciona la extensión cuyo publicador es Microsoft (tiene millones de descargas).
  4. Pulsa Install o Instalar.

Cuidado con imitaciones

Hay extensiones con nombres parecidos. Verifica que el publicador sea Microsoft y que el identificador sea ms-python.python.

Qué debería ocurrir

La extensión se instala en segundos. Puede ofrecerte instalar Pylance (análisis de código); acepta si te lo propone.

Verifica

Comprueba

En la barra inferior de VS Code debería aparecer algo como "Python" o la versión del intérprete cuando abras un archivo .py más adelante. Por ahora confirma que la extensión aparece como instalada en el panel de Extensiones.

7

Crear la carpeta django-lab

Qué vamos a hacer

Crear una carpeta de trabajo llamada django-lab y abrirla en VS Code.

Por qué

Cada proyecto vive en su propia carpeta. Mantener el código organizado desde el primer día evita mezclar archivos de distintos ejercicios.

Código

PowerShell 
cd $HOME\Documents
mkdir django-lab
cd django-lab
code .
CMD 
cd %USERPROFILE%\Documents
mkdir django-lab
cd django-lab
code .
Terminal 
cd ~/Documents
mkdir django-lab
cd django-lab
code .

Dónde va

La carpeta puede estar en Documentos u otra ubicación que prefieras. Lo importante es que recuerdes dónde está. El comando code . abre la carpeta actual en VS Code.

Qué debería ocurrir

VS Code se abre (o se actualiza) mostrando django-lab como carpeta raíz en el explorador de archivos lateral. La terminal integrada debería estar dentro de esa carpeta.

Verifica

Comprueba

En la terminal de VS Code ejecuta pwd (macOS/Linux) o cd sin argumentos (Windows) y confirma que la ruta termina en django-lab.

Error común

"code" no se reconoce — Reinstala VS Code y marca la opción "Add to PATH" durante la instalación. También puedes abrir VS Code manualmente y usar Archivo > Abrir carpeta.

8

Crear el archivo hola.py

Qué vamos a hacer

Crear un script de Python que pida tu nombre y muestre un saludo usando una f-string.

Por qué

Antes de Django necesitas confirmar que puedes escribir, guardar y entender código Python básico: variables, entrada del usuario y salida por pantalla.

Código

django-lab/hola.py
hola.py 
# Mi primer script de Python para Django Lab
nombre = input("¿Cómo te llamas? ")

print(f"Hola, {nombre}. Bienvenido a django-lab.")

Dónde va

En VS Code: clic derecho en el explorador de archivos > Nuevo archivo > escribe hola.py. Pega el código, guarda con Ctrl+S.

Qué hace cada línea

input() muestra un mensaje y espera que escribas texto. Ese texto se guarda en la variable nombre.

print(f"...") es una f-string: las llaves {nombre} insertan el valor de la variable dentro del texto.

Qué debería ocurrir

El archivo aparece en el explorador con icono de Python. El código tiene colores (sintaxis resaltada) gracias a la extensión instalada en el paso 6.

Verifica

Comprueba

Revisa que guardaste el archivo. El punto en la pestaña del archivo desaparece cuando está guardado. La extensión del archivo debe ser .py, no .py.txt.

Error común

El archivo se guardó como hola.py.txt — En Windows, desactiva "Ocultar extensiones de archivo para tipos conocidos" en el Explorador de archivos, o crea el archivo directamente desde VS Code.

9

Ejecutar hola.py

Qué vamos a hacer

Ejecutar el script desde la terminal integrada de VS Code y comprobar la interacción completa.

Por qué

Escribir código que no ejecutas es solo texto decorativo. Este paso cierra el ciclo: escribir, guardar, ejecutar, ver resultado.

Código

terminal (django-lab) 
python hola.py

En macOS/Linux usa python3 hola.py si python no funciona.

Dónde va

Terminal integrada de VS Code, con la carpeta django-lab abierta como proyecto.

Qué debería ocurrir

Salida esperada 
¿Cómo te llamas? Ana
Hola, Ana. Bienvenido a django-lab.

El programa espera que escribas tu nombre y pulses Enter.

Verifica

Comprueba

Ejecuta el script dos veces con nombres distintos. La salida debe cambiar según lo que escribas. Si ves el saludo personalizado, Python y VS Code trabajan juntos correctamente.

Error común

can't open file 'hola.py' — La terminal no está en la carpeta correcta. Ejecuta cd hasta llegar a django-lab o abre la carpeta en VS Code con Archivo > Abrir carpeta.

SyntaxError — Revisa comillas y paréntesis en hola.py. Copia el código del paso 8 exactamente.

10

Checkpoint final

Qué vamos a hacer

Revisar que cumples todos los requisitos antes de pasar al Tutorial 02 sobre entornos virtuales y Django.

Por qué

El siguiente tutorial asume que Python responde en la terminal y que sabes abrir carpetas en VS Code. Si algo falla aquí, fallará todo lo demás.

Lista de verificación final

  • python --version (o python3 --version) muestra Python 3.10+.
  • VS Code abre la carpeta django-lab sin problemas.
  • La extensión Python de Microsoft está instalada.
  • hola.py existe y se ejecuta mostrando el saludo personalizado.
  • Entiendes la diferencia entre escribir código (editor) y ejecutarlo (terminal).

Mensaje de Teacher Jim

Si marcaste todos los puntos, tu base está sólida. En el Tutorial 02 crearemos un entorno virtual aislado e instalaremos Django. No borres la carpeta django-lab; la usaremos como punto de partida.

Verifica

Comprueba

Marca este paso como hecho y pulsa "Marcar como completado" en la barra de progreso lateral. Luego continúa al Tutorial 02.
