Tutorial 03

Tu primera ventana en Django

Hasta ahora preparaste Python, el entorno virtual y Django. Ahora conectas las piezas que hacen funcionar una página web: proyecto, aplicación, URL, vista y template. Al terminar verás tu propio texto en el navegador, no la página de bienvenida por defecto.

Inicial ~50 min 11 pasos

Resultado final

Al terminar este tutorial tendrás:

  • Un proyecto Django llamado config dentro de mi_primer_django.
  • Una aplicación core registrada en INSTALLED_APPS.
  • Una vista home que pasa datos a un template HTML.
  • La URL /home/ mostrando el texto Mi primera página con Django en tu navegador.
mi_primer_django/ ├── .venv/ ├── requirements.txt ├── manage.py ├── config/ │ ├── __init__.py │ ├── settings.py │ ├── urls.py │ ├── asgi.py │ └── wsgi.py └── core/ ├── migrations/ ├── templates/ │ └── core/ │ └── home.html ├── __init__.py ├── admin.py ├── apps.py ├── models.py ├── tests.py ├── urls.py └── views.py

Concepto clave

En Django, el proyecto (config) es la configuración global. La aplicación (core) es donde vive la lógica de una parte concreta de tu sitio. Una petición del navegador recorre URL, vista y template antes de volver como HTML.

Antes de empezar

Confirma que completaste el Tutorial 02 o que cumples estos requisitos:

  • Existe la carpeta mi_primer_django con .venv y requirements.txt.
  • Django 6.0.x está instalado dentro del entorno virtual.
  • django-admin --version responde con 6.0.x cuando el entorno está activo.
  • VS Code está abierto con mi_primer_django como carpeta raíz.
  • Sabes activar el entorno virtual y ves (.venv) en el prompt.

Nota de Teacher Jim

Este tutorial conecta muchas piezas nuevas. No copies archivos sin leer qué hace cada línea. Si algo falla, revisa el paso anterior antes de seguir. La mayoría de errores vienen de un archivo mal nombrado o de olvidar registrar la app en settings.py.

Mapa de la misión

Once pasos en orden. Cada uno construye sobre el anterior.

  1. Activar el entorno virtual en mi_primer_django.
  2. Crear el proyecto Django con django-admin startproject config .
  3. Probar el servidor de desarrollo y ver la página de bienvenida.
  4. Crear la aplicación core con startapp.
  5. Registrar core en INSTALLED_APPS.
  6. Escribir la vista home en core/views.py.
  7. Crear core/urls.py con app_name y la ruta.
  8. Conectar las URLs de core en config/urls.py.
  9. Crear el template core/templates/core/home.html.
  10. Arrancar el servidor de desarrollo.
  11. Verificar que el navegador muestra Mi primera página con Django.
1

Activar el entorno virtual

Qué vamos a hacer

Abrir VS Code en mi_primer_django y activar el entorno virtual antes de ejecutar cualquier comando de Django.

Por qué

Sin el entorno activo, django-admin y manage.py pueden no encontrarse o usar una versión de Django distinta a la que instalaste en el Tutorial 02. Este paso es un recordatorio, no una opción.

Código

PowerShell 
cd $HOME\Documents\mi_primer_django
code .
.\.venv\Scripts\Activate.ps1
CMD 
cd %USERPROFILE%\Documents\mi_primer_django
code .
.venv\Scripts\activate.bat
Terminal 
cd ~/Documents/mi_primer_django
code .
source .venv/bin/activate

Dónde va

Terminal integrada de VS Code, dentro de la carpeta mi_primer_django. Ajusta la ruta si guardaste el proyecto en otra ubicación.

Qué debería ocurrir

El prompt muestra (.venv) al inicio de la línea:

Prompt activado 
(.venv) PS C:\Users\TuUsuario\Documents\mi_primer_django>

Verifica

Comprueba

Ejecuta django-admin --version. Debe mostrar 6.0.x. Si no ves (.venv), no continúes al paso 2.

2

Crear el proyecto con startproject

Qué vamos a hacer

Generar la estructura base del proyecto Django con el nombre config en la carpeta actual.

Por qué

django-admin startproject crea los archivos de configuración que Django necesita: settings.py, urls.py, wsgi.py y el script manage.py. El punto al final (.) indica que quieres crear el proyecto aquí, no en una subcarpeta nueva.

Código

terminal (.venv activo) 
django-admin startproject config .

Qué significa el punto

django-admin startproject config sin punto crea una carpeta config dentro de otra carpeta config. El punto final dice: "usa la carpeta actual (mi_primer_django) como raíz del proyecto". Así manage.py queda al mismo nivel que .venv y requirements.txt.

Dónde va

Terminal con entorno activo, estando en mi_primer_django.

Qué debería ocurrir

Aparecen nuevos archivos y carpetas en el explorador de VS Code:

mi_primer_django/ ├── .venv/ ├── requirements.txt ├── manage.py ← nuevo └── config/ ← nuevo (paquete de configuración) ├── __init__.py ├── settings.py ├── urls.py ├── asgi.py └── wsgi.py

Verifica

Comprueba

Confirma que existe manage.py en la raíz de mi_primer_django y la carpeta config/ con settings.py dentro.

Error común

CommandError: '.\\config' already exists — Ya ejecutaste el comando antes. Si el proyecto está bien, continúa al paso 3. Si quieres empezar de cero, borra manage.py y la carpeta config/ (no borres .venv ni requirements.txt).

3

Probar el servidor de desarrollo

Qué vamos a hacer

Arrancar el servidor de desarrollo de Django y abrir la página de bienvenida por defecto en el navegador.

Por qué

Antes de escribir tu propia página, confirmas que el proyecto funciona. manage.py es el asistente de Django para este proyecto: migraciones, servidor, creación de apps y más.

Código

terminal (.venv activo) 
python manage.py runserver

Dónde va

Terminal con entorno activo, en la carpeta que contiene manage.py.

Qué debería ocurrir

La terminal muestra algo como:

Salida del servidor 
Watching for file changes with StatReloader
Performing system checks...

System check identified no issues (0 silenced).

You have 18 unapplied migration(s)...
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.

Abre http://127.0.0.1:8000/ en tu navegador. Verás la página de cohete de Django con el mensaje "The install worked successfully!".

Detén el servidor con Ctrl+C (o Ctrl+Break en Windows) antes de continuar al paso 4.

Verifica

Comprueba

La página de bienvenida de Django carga sin error 404 ni pantalla en blanco. El aviso de migraciones pendientes es normal; las aplicaremos en tutoriales posteriores.

Importante

Solo puedes tener un servidor escuchando en el puerto 8000. Si ves "That port is already in use", cierra la terminal anterior o usa python manage.py runserver 8001.

4

Crear la aplicación core

Qué vamos a hacer

Crear una aplicación Django llamada core que contendrá nuestra primera vista y template.

Por qué

Un proyecto es el contenedor global: configuración, URLs principales, base de datos. Una aplicación es un módulo reutilizable con una responsabilidad concreta. Un proyecto puede tener muchas apps (core, blog, tienda). Aquí core aloja la página de inicio.

Proyecto vs aplicación

Proyecto (config) — La configuración del sitio completo. Un solo proyecto por carpeta de trabajo.

Aplicación (core) — Código de una funcionalidad. Puedes tener varias apps en un proyecto o reutilizar una app en otro proyecto.

Código

terminal (.venv activo) 
python manage.py startapp core

Dónde va

Terminal en la raíz del proyecto, junto a manage.py.

Qué debería ocurrir

Se crea la carpeta core/ con esta estructura:

mi_primer_django/ ├── manage.py ├── config/ └── core/ ← nueva aplicación ├── migrations/ │ └── __init__.py ├── __init__.py ├── admin.py ├── apps.py ├── models.py ├── tests.py └── views.py

Verifica

Comprueba

El explorador de VS Code muestra la carpeta core al mismo nivel que config, no dentro de ella.

5

Registrar core en INSTALLED_APPS

Qué vamos a hacer

Decirle a Django que la aplicación core forma parte de este proyecto añadiéndola a INSTALLED_APPS.

Por qué

Django solo carga apps registradas. Sin este paso, tus vistas, templates y modelos de core no serán encontrados. Es uno de los olvidos más frecuentes de principiantes.

Código

config/settings.py
settings.py 
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'core',
]

Añade 'core', al final de la lista. No borres las apps que Django trae por defecto.

Dónde va

Archivo config/settings.py, busca la variable INSTALLED_APPS cerca del inicio del archivo.

Qué debería ocurrir

Al guardar el archivo, no aparece ningún error en la terminal si el servidor estaba corriendo (se recarga solo). Si ejecutas python manage.py check, debe reportar cero problemas.

Verifica

Comprueba

La cadena es exactamente 'core' (minúsculas, sin espacios extra). Django distingue mayúsculas y minúsculas en los nombres de apps.

Error común

TemplateDoesNotExist más adelante — A menudo la app no está en INSTALLED_APPS. Vuelve a este paso antes de buscar otros problemas.

6

Escribir la vista home en views.py

Qué vamos a hacer

Crear una función vista llamada home que prepara datos y los envía a un template HTML.

Por qué

La vista es el cerebro de cada página: recibe la petición, decide qué datos necesita y devuelve una respuesta. Aquí usamos render, que combina un template con un diccionario de contexto.

Código

core/views.py
views.py 
from django.shortcuts import render


def home(request):
    contexto = {
        'titulo': 'Mi primera página con Django',
        'mensaje': 'Si ves este texto, tu proyecto, app, URL, vista y template están conectados.',
        'autor': 'Teacher Jim',
    }
    return render(request, 'core/home.html', contexto)

Qué hace cada parte

request — Objeto con la información de la petición del navegador (siempre el primer argumento).

contexto — Diccionario con variables que el template podrá usar.

render(...) — Busca el template, le pasa el contexto y devuelve HTML listo para el navegador.

Dónde va

Reemplaza el contenido de core/views.py. El archivo viene casi vacío después de startapp.

Qué debería ocurrir

El archivo guarda sin errores de sintaxis. VS Code no subraya en rojo las importaciones de Django si el intérprete del entorno virtual está seleccionado.

Verifica

Comprueba

La ruta del template en render es 'core/home.html'. La crearemos en el paso 9 siguiendo la convención templates/core/.

7

Crear core/urls.py con app_name

Qué vamos a hacer

Crear un archivo de URLs propio de la app core que conecta la ruta /home/ con la vista home.

Por qué

Cada app puede tener sus propias URLs. app_name = 'core' define un espacio de nombres para evitar colisiones si otra app también tuviera una ruta llamada home. Más adelante podrás referenciar esta URL como core:home.

Código

core/urls.py
urls.py (nuevo archivo) 
from django.urls import path

from . import views

app_name = 'core'

urlpatterns = [
    path('home/', views.home, name='home'),
]

Dónde va

Crea el archivo core/urls.py al mismo nivel que views.py. En VS Code: clic derecho en la carpeta core > Nuevo archivo > urls.py.

Qué debería ocurrir

El archivo existe junto a los demás módulos de core:

core/ ├── urls.py ← nuevo ├── views.py ├── models.py └── ...

Verifica

Comprueba

La importación es from . import views (punto = mismo paquete). El name='home' identifica esta ruta dentro del espacio core.

8

Conectar las URLs en config/urls.py

Qué vamos a hacer

Incluir las URLs de core en el enrutador principal del proyecto con include.

Por qué

Cuando el navegador pide una URL, Django primero consulta config/urls.py. Sin include, Django no sabe que existen las rutas definidas en core/urls.py.

Código

config/urls.py
urls.py 
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('core.urls')),
]

Añade include a la importación y la línea path('', include('core.urls')). La cadena vacía '' significa que las rutas de core se montan en la raíz del sitio, así home/ será accesible como /home/.

Dónde va

Archivo config/urls.py, dentro de la lista urlpatterns.

Qué debería ocurrir

La ruta completa queda así: petición a http://127.0.0.1:8000/home/ → config/urls.py delega a core/urls.py → ejecuta views.home.

Verifica

Comprueba

Guarda el archivo. Si el servidor está corriendo, verás "Watching for file changes" en la terminal al recargar.

Error común

404 Page not found en /home/ — Revisa que include('core.urls') esté escrito exactamente así y que core/urls.py exista. Un error tipográfico en el nombre del módulo rompe toda la ruta.

9

Crear el template home.html

Qué vamos a hacer

Crear un archivo HTML completo que muestra las variables del contexto que definimos en la vista.

Por qué

El template es la capa de presentación. Separa el diseño (HTML) de la lógica (Python en la vista). Django usa su propio lenguaje de plantillas con {{ variable }} para insertar datos.

Código

core/templates/core/home.html

Crea las carpetas templates y core dentro de la app si no existen. La ruta completa es importante: Django busca core/home.html dentro de cualquier carpeta templates/ de apps registradas.

home.html 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{{ titulo }}</title>
  <style>
    body {
      font-family: system-ui, sans-serif;
      max-width: 40rem;
      margin: 3rem auto;
      padding: 0 1.5rem;
      line-height: 1.6;
      color: #1a1a2e;
      background: #f8f9fc;
    }
    h1 {
      color: #0d47a1;
      border-bottom: 3px solid #90caf9;
      padding-bottom: 0.5rem;
    }
    .mensaje {
      background: #e3f2fd;
      border-left: 4px solid #1976d2;
      padding: 1rem 1.25rem;
      border-radius: 0 8px 8px 0;
    }
    footer {
      margin-top: 2rem;
      font-size: 0.9rem;
      color: #546e7a;
    }
  </style>
</head>
<body>
  <h1>{{ titulo }}</h1>
  <p class="mensaje">{{ mensaje }}</p>
  <footer>
    <p>Creado siguiendo los tutoriales de {{ autor }}.</p>
  </footer>
</body>
</html>

Dónde va

Dentro de la app core, no en la raíz del proyecto:

core/ ├── templates/ │ └── core/ │ └── home.html ← aquí ├── urls.py └── views.py

Qué debería ocurrir

El archivo guarda correctamente. Las llaves dobles {{ titulo }} no se muestran en el navegador: Django las reemplaza por el valor del contexto.

Verifica

Comprueba

La ruta del archivo coincide con lo que pide render: 'core/home.html' busca templates/core/home.html dentro de la app.

Error común

TemplateDoesNotExist: core/home.html — El archivo está en la carpeta equivocada. Debe estar en core/templates/core/, no en config/templates/ ni directamente en core/home.html.

10

Arrancar el servidor de desarrollo

Qué vamos a hacer

Iniciar de nuevo el servidor para probar la página completa con URL, vista y template conectados.

Por qué

Todos los archivos ya están en su sitio. El servidor de desarrollo recarga automáticamente cuando guardas cambios, pero arrancarlo de nuevo confirma que no hay errores de importación ni de sintaxis al inicio.

Código

terminal (.venv activo) 
python manage.py runserver

Dónde va

Terminal en la raíz del proyecto, con entorno virtual activo.

Qué debería ocurrir

El servidor arranca sin errores de importación. La terminal queda esperando peticiones:

Servidor listo 
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.

Verifica

Comprueba

Si aparece un traceback en rojo al arrancar, léelo desde abajo: la última línea suele indicar el archivo y la línea con el error. Corrige antes de ir al paso 11.

11

Verificar Mi primera página con Django

Qué vamos a hacer

Abrir la URL correcta en el navegador y confirmar que ves el título y el mensaje de tu template, no la página de cohete de Django.

Por qué

La verificación final cierra el ciclo. Si el texto aparece, significa que las cinco piezas —proyecto, app, URL, vista y template— están conectadas correctamente.

Código

Abre esta URL en tu navegador (con el servidor corriendo):

Navegador 
http://127.0.0.1:8000/home/

Presta atención a la barra final: /home/ con la diagonal al final. Django suele redirigir si falta, pero usa la forma canónica.

Qué debería ocurrir

El navegador muestra una página con:

  • Un encabezado Mi primera página con Django (valor de {{ titulo }}).
  • El párrafo del mensaje en un recuadro azul claro.
  • El pie de página con "Creado siguiendo los tutoriales de Teacher Jim".

En la terminal del servidor verás una línea como:

Log del servidor 
[07/Jul/2026 13:45:00] "GET /home/ HTTP/1.1" 200 892

El código 200 significa que la petición fue exitosa.

Lista de verificación final

  • El entorno virtual está activo cuando trabajas en el proyecto.
  • Existe manage.py y la carpeta config/.
  • La app core está en INSTALLED_APPS.
  • core/views.py define la función home con contexto.
  • core/urls.py mapea home/ a la vista.
  • config/urls.py incluye core.urls.
  • El template core/templates/core/home.html existe.
  • El navegador en /home/ muestra Mi primera página con Django.

Verifica

Comprueba

Marca este paso, pulsa "Marcar como completado" y continúa al Tutorial 04 cuando estés listo.

Si algo falla

404 en /home/ — Revisa pasos 7 y 8 (URLs).

TemplateDoesNotExist — Revisa pasos 5 y 9 (app registrada y ruta del template).

Página en blanco o error 500 — Mira la terminal del servidor; el traceback indica el archivo problemático.

Flujo de una petición en Django

Cuando visitas http://127.0.0.1:8000/home/, Django sigue este recorrido. Memorizarlo te ahorrará horas de depuración:

URL del navegador GET /home/ config/urls.py include('core.urls') core/urls.py path('home/', views.home) core/views.py render con contexto core/templates/core/home.html HTML generado Navegador

Resumen del flujo

URL — Lo que el usuario escribe o pulsa.

urls.py — Tabla de enrutamiento: qué vista responde a cada ruta.

views.py — Lógica: prepara datos y elige el template.

template — Presentación: HTML con variables del contexto.

Navegador — Recibe el HTML y lo muestra al usuario.

Mini reto

Prueba que entiendes el flujo sin mirar los pasos anteriores:

  1. Añade una variable 'fecha' al contexto en views.py con el valor 'Julio 2026'.
  2. Muestra esa variable en el template debajo del mensaje principal con {{ fecha }}.
  3. Crea una segunda ruta acerca/ en core/urls.py que apunte a una nueva vista acerca con su propio template.
  4. Verifica ambas URLs en el navegador sin reiniciar el servidor (el autoreload lo hace por ti).

Si puedes añadir una ruta nueva y verla en el navegador, ya no solo copias tutoriales: estás pensando como desarrollador Django.
