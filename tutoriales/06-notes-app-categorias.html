Tutorial 06 · Fase 1 · Mini Proyecto 01

Notes App con categorías

Vas a construir una aplicación privada de notas. Cada usuario solo ve las suyas. Aprenderás ForeignKey, filtros, búsqueda, favoritos y tarjetas en plantillas. Este proyecto refuerza lo del To-Do y lo deja listo para portafolio.

Fase 1 2 clases 12 pasos Mini proyecto

Resultado final

Al terminar tendrás una app notes con:

  • Modelos Category y Note ligados al usuario.
  • CRUD de notas protegido con @login_required.
  • Categorías con color, favoritos, búsqueda y filtros.
  • Plantillas con cards y un CSS claro para mostrar en captura.

Por qué sirve al portafolio

Un reclutador entiende al instante: autenticación, datos propios por usuario, relaciones ForeignKey y filtros reales. No es un hola mundo.

mi_primer_django/ ├── notes/ │ ├── models.py │ ├── forms.py │ ├── views.py │ ├── urls.py │ ├── admin.py │ └── templates/notes/ │ ├── note_list.html │ ├── note_form.html │ ├── note_detail.html │ ├── note_confirm_delete.html │ └── category_form.html ├── static/css/notes.css └── ...

Antes de empezar

  • Completaste el Tutorial 04 (To-Do + Login) o tienes login funcionando.
  • Existe templates/base.html y autenticación con django.contrib.auth.
  • El entorno virtual está activo y Django 6.0.x responde.
  • VS Code abierto en la carpeta que contiene manage.py.

Nota de Teacher Jim

Reutiliza el login del Tutorial 04. No reinventes autenticación: concentra energía en modelos, filtros y plantillas.

Arquitectura

URL /notes/ → vistas de notes → modelos filtrados por request.user → templates con cards. El usuario nunca ve notas ajenas porque cada consulta lleva filter(user=request.user) o get_object_or_404(..., user=request.user).

2

Definir Category y Note

Qué vamos a hacer

Escribir los modelos Category y Note con ForeignKey al usuario y color opcional en categorías.

Por qué

Category organiza. Note guarda el contenido. El ForeignKey a usuario aísla datos. SET_NULL en categoría evita borrar notas si eliminas una categoría.

Código

notes/models.py 
from django.conf import settings
from django.db import models


class Category(models.Model):
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name="note_categories",
    )
    name = models.CharField(max_length=80)
    color = models.CharField(max_length=20, default="#7DFF58")

    class Meta:
        ordering = ["name"]
        verbose_name_plural = "categories"

    def __str__(self):
        return self.name


class Note(models.Model):
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name="notes",
    )
    category = models.ForeignKey(
        Category,
        on_delete=models.SET_NULL,
        null=True,
        blank=True,
        related_name="notes",
    )
    title = models.CharField(max_length=150)
    content = models.TextField()
    is_favorite = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        ordering = ["-updated_at"]

    def __str__(self):
        return self.title

Reemplaza el contenido de notes/models.py.

Verifica

Comprueba

5

Formularios NoteForm y CategoryForm

Qué vamos a hacer

Crear forms.py con ModelForms. El queryset de categorías se limita al usuario autenticado.

Por qué

Sin filtrar categorías, un usuario vería categorías ajenas en el desplegable. Eso es un fallo de seguridad de datos.

Código

notes/forms.py 
from django import forms
from .models import Category, Note


class CategoryForm(forms.ModelForm):
    class Meta:
        model = Category
        fields = ["name", "color"]
        widgets = {
            "name": forms.TextInput(attrs={"placeholder": "Nombre de la categoría"}),
            "color": forms.TextInput(attrs={"type": "color"}),
        }


class NoteForm(forms.ModelForm):
    class Meta:
        model = Note
        fields = ["title", "content", "category", "is_favorite"]
        widgets = {
            "title": forms.TextInput(attrs={"placeholder": "Título de la nota"}),
            "content": forms.Textarea(attrs={"rows": 8, "placeholder": "Escribe tu nota..."}),
        }

    def __init__(self, *args, user=None, **kwargs):
        super().__init__(*args, **kwargs)
        self.user = user
        if user is not None:
            self.fields["category"].queryset = Category.objects.filter(user=user)
            self.fields["category"].required = False

Dónde va

Archivo nuevo notes/forms.py.

Qué debería ocurrir

Puedes importar los formularios desde las vistas sin error.

Verifica

Comprueba

El constructor de NoteForm recibe user= y filtra category.queryset.

6

Vistas con login_required y filtro por usuario

Qué vamos a hacer

Implementar listado, CRUD, detalle, toggle de favorito y creación de categoría. Toda consulta filtra por request.user.

Por qué

Esto es el corazón del proyecto. URL → View → Model filtrado → Template. Sin el filtro de usuario, tendrías una app insegura.

Código

notes/views.py 
from django.contrib import messages
from django.contrib.auth.decorators import login_required
from django.db.models import Q
from django.shortcuts import get_object_or_404, redirect, render
from .forms import CategoryForm, NoteForm
from .models import Category, Note


@login_required
def note_list(request):
    notes = Note.objects.filter(user=request.user).select_related("category")
    categories = Category.objects.filter(user=request.user)

    q = request.GET.get("q", "").strip()
    category_id = request.GET.get("category", "")
    favorites_only = request.GET.get("favorites") == "1"

    if q:
        notes = notes.filter(Q(title__icontains=q) | Q(content__icontains=q))
    if category_id.isdigit():
        notes = notes.filter(category_id=int(category_id))
    if favorites_only:
        notes = notes.filter(is_favorite=True)

    context = {
        "notes": notes,
        "categories": categories,
        "q": q,
        "category_id": category_id,
        "favorites_only": favorites_only,
        "total": notes.count(),
    }
    return render(request, "notes/note_list.html", context)


@login_required
def note_create(request):
    if request.method == "POST":
        form = NoteForm(request.POST, user=request.user)
        if form.is_valid():
            note = form.save(commit=False)
            note.user = request.user
            note.save()
            messages.success(request, "Nota creada.")
            return redirect("notes:note_list")
    else:
        form = NoteForm(user=request.user)
    return render(request, "notes/note_form.html", {"form": form, "title": "Nueva nota"})


@login_required
def note_edit(request, pk):
    note = get_object_or_404(Note, pk=pk, user=request.user)
    if request.method == "POST":
        form = NoteForm(request.POST, instance=note, user=request.user)
        if form.is_valid():
            form.save()
            messages.success(request, "Nota actualizada.")
            return redirect("notes:note_list")
    else:
        form = NoteForm(instance=note, user=request.user)
    return render(request, "notes/note_form.html", {"form": form, "title": "Editar nota"})


@login_required
def note_delete(request, pk):
    note = get_object_or_404(Note, pk=pk, user=request.user)
    if request.method == "POST":
        note.delete()
        messages.success(request, "Nota eliminada.")
        return redirect("notes:note_list")
    return render(request, "notes/note_confirm_delete.html", {"note": note})


@login_required
def note_detail(request, pk):
    note = get_object_or_404(Note, pk=pk, user=request.user)
    return render(request, "notes/note_detail.html", {"note": note})


@login_required
def toggle_favorite(request, pk):
    note = get_object_or_404(Note, pk=pk, user=request.user)
    if request.method == "POST":
        note.is_favorite = not note.is_favorite
        note.save(update_fields=["is_favorite", "updated_at"])
    return redirect(request.POST.get("next") or "notes:note_list")


@login_required
def category_create(request):
    if request.method == "POST":
        form = CategoryForm(request.POST)
        if form.is_valid():
            category = form.save(commit=False)
            category.user = request.user
            category.save()
            messages.success(request, "Categoría creada.")
            return redirect("notes:note_list")
    else:
        form = CategoryForm()
    return render(request, "notes/category_form.html", {"form": form})

Dónde va

Reemplaza notes/views.py.

Qué debería ocurrir

Las vistas están listas, pero faltan URLs y templates para verlas en el navegador.

Verifica

Comprueba

Cada get_object_or_404 incluye user=request.user. La lista usa Q para buscar en título y contenido.

Error común

forgot to pass user= to NoteForm — El desplegable de categorías fallará o mostrará todas. Pasa siempre user=request.user.

7

URLs de la app y del proyecto

Qué vamos a hacer

Crear notes/urls.py con app_name e incluirlo en config/urls.py bajo el prefijo notes/.

Por qué

Sin rutas, las vistas no existen para el navegador. El namespace notes: evita colisiones con tasks:.

Código

notes/urls.py 
from django.urls import path
from . import views

app_name = "notes"

urlpatterns = [
    path("", views.note_list, name="note_list"),
    path("nueva/", views.note_create, name="note_create"),
    path("<int:pk>/", views.note_detail, name="note_detail"),
    path("<int:pk>/editar/", views.note_edit, name="note_edit"),
    path("<int:pk>/eliminar/", views.note_delete, name="note_delete"),
    path("<int:pk>/favorito/", views.toggle_favorite, name="toggle_favorite"),
    path("categorias/nueva/", views.category_create, name="category_create"),
]
config/urls.py 
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path("admin/", admin.site.urls),
    path("accounts/", include("django.contrib.auth.urls")),
    path("tasks/", include("tasks.urls")),
    path("notes/", include("notes.urls")),
    path("", include("core.urls")),
]

Dónde va

Archivo nuevo en la app; en config, añade la línea path("notes/", ...) respetando tus includes existentes (tasks, accounts, core).

Qué debería ocurrir

Django conoce las rutas. Al abrirlas sin templates verás TemplateDoesNotExist — eso es el siguiente paso.

Verifica

Comprueba

python manage.py show_urls no existe por defecto; usa python manage.py check y confirma cero issues.

Errores comunes de este proyecto

  • Notas ajenas visibles — Olvidaste filter(user=request.user) o el user= en get_object_or_404.
  • Categorías de otros usuarios en el select — No pasaste user=request.user a NoteForm.
  • TemplateDoesNotExist: notes/note_list.html — La carpeta debe ser notes/templates/notes/, no solo templates/.
  • NoReverseMatch notes:... — Falta app_name = "notes" o el include en config/urls.py.
  • IntegrityError user_id — Guardaste el form sin commit=False y sin asignar note.user.
  • Color no se ve — El CSS usa style="--accent: ..."; verifica que notes.css esté cargado.
