Tutorial 08 · Fase 2 · Proyecto Guiado

Mini CRM de Contactos

Construyes un directorio privado: crear, editar, eliminar y buscar contactos; filtrar por tipo y estado; ver un dashboard con conteos. Es el salto de mini proyecto a algo que parece herramienta de trabajo.

Fase 2 2 clases 12 pasos Proyecto guiado

Resultado final

  • Modelo Contact con CONTACT_TYPES y STATUS_CHOICES.
  • CRUD completo aislado por usuario.
  • Búsqueda con Q (nombre, correo, empresa).
  • Filtros por tipo y estado + dashboard con agregaciones Count.

Por qué Fase 2

Aquí demuestras choices, agregaciones y un listado profesional. Es evidencia clara de que ya no solo haces CRUD plano.

contacts/ ├── models.py ├── forms.py ├── views.py ├── urls.py ├── admin.py └── templates/contacts/ ├── contact_list.html ├── contact_form.html ├── contact_detail.html ├── contact_confirm_delete.html └── dashboard.html

Antes de empezar

  • Login del Tutorial 04 funcionando.
  • Completaste Notes (06) o entiendes ForeignKey + login_required.
  • Entorno virtual activo.

Nota de Teacher Jim

No copies el modelo a medias. CONTACT_TYPES y STATUS_CHOICES deben coincidir exactamente: son parte del contrato del proyecto.

Arquitectura

/contacts/ lista y filtra. /contacts/dashboard/ agrega. El detalle y los formularios reutilizan el mismo aislamiento por request.user que Notes y Tasks.

Modelo Contact con choices

Qué vamos a hacer

Definir el modelo con tipos, estados y campos de seguimiento.

Por qué

choices documenta el dominio en el modelo y genera labels con get_FOO_display.

Código

contacts/models.py 
from django.conf import settings
from django.db import models


class Contact(models.Model):
    CONTACT_TYPES = [
        ("student", "Estudiante"),
        ("teacher", "Docente"),
        ("company", "Empresa"),
        ("client", "Cliente"),
        ("other", "Otro"),
    ]

    STATUS_CHOICES = [
        ("new", "Nuevo"),
        ("contacted", "Contactado"),
        ("pending", "Pendiente"),
        ("closed", "Cerrado"),
    ]

    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name="contacts",
    )
    full_name = models.CharField(max_length=150)
    email = models.EmailField(blank=True)
    phone = models.CharField(max_length=30, blank=True)
    company = models.CharField(max_length=120, blank=True)
    contact_type = models.CharField(max_length=20, choices=CONTACT_TYPES)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default="new")
    notes = models.TextField(blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ["full_name"]

    def __str__(self):
        return self.full_name

5

Vistas CRUD + aislamiento

Qué vamos a hacer

Implementar listado, crear, editar, eliminar, detalle y dashboard.

Por qué

Un CRM sin aislamiento de usuarios no es demostrable en entrevista.

Código

contacts/views.py 
from django.contrib import messages
from django.contrib.auth.decorators import login_required
from django.db.models import Count, Q
from django.shortcuts import get_object_or_404, redirect, render
from .forms import ContactForm
from .models import Contact


def _base_qs(user):
    return Contact.objects.filter(user=user)


@login_required
def contact_list(request):
    contacts = _base_qs(request.user)
    q = request.GET.get("q", "").strip()
    contact_type = request.GET.get("type", "")
    status = request.GET.get("status", "")

    if q:
        contacts = contacts.filter(
            Q(full_name__icontains=q)
            | Q(email__icontains=q)
            | Q(company__icontains=q)
        )
    if contact_type:
        contacts = contacts.filter(contact_type=contact_type)
    if status:
        contacts = contacts.filter(status=status)

    context = {
        "contacts": contacts,
        "q": q,
        "contact_type": contact_type,
        "status": status,
        "type_choices": Contact.CONTACT_TYPES,
        "status_choices": Contact.STATUS_CHOICES,
        "total": contacts.count(),
    }
    return render(request, "contacts/contact_list.html", context)


@login_required
def contact_create(request):
    if request.method == "POST":
        form = ContactForm(request.POST)
        if form.is_valid():
            contact = form.save(commit=False)
            contact.user = request.user
            contact.save()
            messages.success(request, "Contacto creado.")
            return redirect("contacts:contact_list")
    else:
        form = ContactForm()
    return render(request, "contacts/contact_form.html", {"form": form, "title": "Nuevo contacto"})


@login_required
def contact_edit(request, pk):
    contact = get_object_or_404(Contact, pk=pk, user=request.user)
    if request.method == "POST":
        form = ContactForm(request.POST, instance=contact)
        if form.is_valid():
            form.save()
            messages.success(request, "Contacto actualizado.")
            return redirect("contacts:contact_list")
    else:
        form = ContactForm(instance=contact)
    return render(request, "contacts/contact_form.html", {"form": form, "title": "Editar contacto"})


@login_required
def contact_delete(request, pk):
    contact = get_object_or_404(Contact, pk=pk, user=request.user)
    if request.method == "POST":
        contact.delete()
        messages.success(request, "Contacto eliminado.")
        return redirect("contacts:contact_list")
    return render(request, "contacts/contact_confirm_delete.html", {"contact": contact})


@login_required
def contact_detail(request, pk):
    contact = get_object_or_404(Contact, pk=pk, user=request.user)
    return render(request, "contacts/contact_detail.html", {"contact": contact})


@login_required
def dashboard(request):
    qs = _base_qs(request.user)
    status_labels = dict(Contact.STATUS_CHOICES)
    type_labels = dict(Contact.CONTACT_TYPES)
    by_status = [
        {"label": status_labels.get(r["status"], r["status"]), "total": r["total"]}
        for r in qs.values("status").annotate(total=Count("id")).order_by("status")
    ]
    by_type = [
        {"label": type_labels.get(r["contact_type"], r["contact_type"]), "total": r["total"]}
        for r in qs.values("contact_type").annotate(total=Count("id")).order_by("contact_type")
    ]
    context = {
        "total": qs.count(),
        "by_status": by_status,
        "by_type": by_type,
        "recent": qs.order_by("-created_at")[:5],
    }
    return render(request, "contacts/dashboard.html", context)

Errores comunes

  • NoReverseMatch contacts:dashboard — Include faltante o app_name incorrecto.
  • FieldError en Q — Typo en nombre de campo (full_name vs name).
  • IntegrityError user_id — Olvidaste assignar contact.user antes de save.
  • Dashboard vacío con datos en lista — Estás mirando otro usuario o no recargaste tras crear.
  • Choices no muestran label — Usa get_status_display, no el valor crudo new.

Mini retos

Reto A: Ordenar por -created_at o nombre mediante ?sort=.

Reto B: Añadir campo last_contacted DateField opcional y mostrarlo en detalle.

Reto C: Exportar el listado filtrado a CSV simple con una vista que devuelva text/csv.
