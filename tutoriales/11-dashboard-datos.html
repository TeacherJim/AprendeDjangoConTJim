Proyecto profesional: Dashboard de gastos con Chart.js

Registras ingresos y gastos, calculas totales y balance, y dibujas gráficos por categoría y por mes. Django prepara los datos; Chart.js los pinta en el navegador.

Proyecto profesional Fase 3 4 clases 10 pasos

Resultado final

  • Modelo Transaction (ingreso/gasto) con monto Decimal, categoría y fecha.
  • Totales de ingresos, gastos y balance por usuario.
  • Gráficos Chart.js: por categoría y por mes.
  • Datos pasados con json_script (seguro frente a XSS).
  • Tabla filtrable por tipo y mes.

Backend vs frontend

Django agrega y serializa. Chart.js solo recibe JSON y dibuja. No construyas API REST para este proyecto.

Nota de Teacher Jim

Chart.js desde CDN está bien en tu app Django local para aprender. Tu portafolio en GitHub Pages es otra cosa: ahí solo muestras capturas/enlaces del proyecto, no ejecutas Django ni necesitas Chart.js en el portal educativo.

Antes de empezar

  • Dominas CRUD con login_required y ForeignKey a User.
  • Completaste el HelpDesk o un proyecto similar de Fase 2/3.
  • Tienes cuenta con login funcionando (puedes reutilizar auth del To-Do).

Modelo Transaction

Qué vamos a hacer

Definir ingresos y gastos con DecimalField (nunca float para dinero).

finance/models.py 
from django.conf import settings
from django.db import models


class Transaction(models.Model):
    TYPE_CHOICES = [
        ("income", "Ingreso"),
        ("expense", "Gasto"),
    ]

    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.CASCADE,
        related_name="transactions",
    )
    transaction_type = models.CharField(max_length=20, choices=TYPE_CHOICES)
    category = models.CharField(max_length=100)
    amount = models.DecimalField(max_digits=10, decimal_places=2)
    date = models.DateField()
    description = models.CharField(max_length=200, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ["-date", "-created_at"]

    def __str__(self):
        sign = "+" if self.transaction_type == "income" else "-"
        return f"{sign}{self.amount} {self.category}"

¿Por qué Decimal?

Los float en binario no representan 0.1 exacto. Para dinero usamos Decimal. Luego, al pasar a JavaScript, convertiremos a string o float de forma consciente.

Vistas: totales, filtros y datos para gráficos

Qué vamos a hacer

Calcular suma de ingresos/gastos, balance y series para Chart.js.

finance/views.py 
from decimal import Decimal
from collections import defaultdict

from django.contrib.auth.decorators import login_required
from django.db.models import Sum, Q
from django.db.models.functions import TruncMonth
from django.shortcuts import redirect, render
from django.contrib import messages

from .forms import TransactionForm
from .models import Transaction


def _dec(value):
    """Convierte Decimal/None a float seguro para JSON de gráficos."""
    if value is None:
        return 0.0
    if isinstance(value, Decimal):
        return float(value)
    return float(value)


@login_required
def dashboard(request):
    qs = Transaction.objects.filter(user=request.user)

    month = request.GET.get("month", "")  # formato YYYY-MM
    if month:
        year, mon = month.split("-")
        qs = qs.filter(date__year=int(year), date__month=int(mon))

    aggregates = qs.aggregate(
        income=Sum("amount", filter=Q(transaction_type="income")),
        expense=Sum("amount", filter=Q(transaction_type="expense")),
    )
    income = aggregates["income"] or Decimal("0")
    expense = aggregates["expense"] or Decimal("0")
    balance = income - expense

    # Gastos por categoría
    by_cat = (
        qs.filter(transaction_type="expense")
        .values("category")
        .annotate(total=Sum("amount"))
        .order_by("-total")
    )
    category_labels = [row["category"] for row in by_cat]
    category_values = [_dec(row["total"]) for row in by_cat]

    # Totales por mes (ingresos y gastos)
    monthly = (
        Transaction.objects.filter(user=request.user)
        .annotate(month=TruncMonth("date"))
        .values("month", "transaction_type")
        .annotate(total=Sum("amount"))
        .order_by("month")
    )
    month_map = defaultdict(lambda: {"income": 0.0, "expense": 0.0})
    for row in monthly:
        key = row["month"].strftime("%Y-%m")
        month_map[key][row["transaction_type"]] = _dec(row["total"])
    month_labels = list(month_map.keys())
    month_income = [month_map[m]["income"] for m in month_labels]
    month_expense = [month_map[m]["expense"] for m in month_labels]

    chart_payload = {
        "category": {"labels": category_labels, "values": category_values},
        "monthly": {
            "labels": month_labels,
            "income": month_income,
            "expense": month_expense,
        },
    }

    return render(request, "finance/dashboard.html", {
        "income": income,
        "expense": expense,
        "balance": balance,
        "transactions": qs[:30],
        "month": month,
        # dict listo para {{ chart_data|json_script:"chart-data" }}
        "chart_data": chart_payload,
    })


@login_required
def transaction_create(request):
    if request.method == "POST":
        form = TransactionForm(request.POST)
        if form.is_valid():
            obj = form.save(commit=False)
            obj.user = request.user
            obj.save()
            messages.success(request, "Movimiento registrado.")
            return redirect("finance:dashboard")
    else:
        form = TransactionForm()
    return render(request, "finance/transaction_form.html", {"form": form})


@login_required
def transaction_list(request):
    qs = Transaction.objects.filter(user=request.user)
    t = request.GET.get("type", "")
    if t in ("income", "expense"):
        qs = qs.filter(transaction_type=t)
    return render(request, "finance/transaction_list.html", {
        "transactions": qs,
        "type": t,
    })

Nota de Teacher Jim

Mira la función _dec: JSON no tiene tipo Decimal. Si serializas mal, verás TypeError: Object of type Decimal is not JSON serializable. Convertir a float (o a string) es el paso obligatorio.

6

Pasar JSON al template con json_script

Qué vamos a hacer

Explicar por qué usamos json_script en lugar de meter JSON crudo en un <script>.

Por qué

json_script escapa caracteres peligrosos (<, etc.) y evita que un texto malicioso cierre tu script. Es la forma recomendada por Django.

Fragmento en dashboard.html 
{{ chart_data|json_script:"chart-data" }}

La vista pasa un dict llamado chart_data (no un string). El filtro json_script lo serializa de forma segura a un <script type="application/json" id="chart-data">. Tu charts.js lo lee con JSON.parse.

Decimal y JSON

Si dejaste un Decimal dentro del dict, Django/JSON fallará. Por eso _dec() convierte a float antes de armar chart_payload. Alternativa válida: guardar strings (str(amount)) y parsear en JS.

static/js/charts.js 
(function () {
  const el = document.getElementById("chart-data");
  if (!el) return;
  const data = JSON.parse(el.textContent);

  const cat = document.getElementById("chartCategory");
  if (cat && data.category) {
    new Chart(cat, {
      type: "doughnut",
      data: {
        labels: data.category.labels,
        datasets: [{
          data: data.category.values,
          backgroundColor: [
            "#38bdf8", "#818cf8", "#34d399", "#fbbf24", "#f87171", "#a78bfa", "#2dd4bf"
          ],
        }],
      },
      options: { plugins: { legend: { position: "bottom" } } },
    });
  }

  const mon = document.getElementById("chartMonthly");
  if (mon && data.monthly) {
    new Chart(mon, {
      type: "bar",
      data: {
        labels: data.monthly.labels,
        datasets: [
          { label: "Ingresos", data: data.monthly.income, backgroundColor: "#34d399" },
          { label: "Gastos", data: data.monthly.expense, backgroundColor: "#f87171" },
        ],
      },
      options: {
        scales: { y: { beginAtZero: true } },
        plugins: { legend: { position: "bottom" } },
      },
    });
  }
})();

Errores comunes

Error común

TypeError: Decimal is not JSON serializable — Convierte con float() o serializa strings. Usa _dec.

Error común

Chart is not defined — El script de Chart.js CDN debe cargarse antes de charts.js.

Error común

Gráficos vacíos — No hay gastos, o filtraste un mes sin datos. Quita el filtro y verifica el JSON en el inspector (id chart-data).
