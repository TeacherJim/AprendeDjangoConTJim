Proyecto final: Portafolio profesional en GitHub Pages

Construyes una página personal estática (HTML/CSS/JS) que muestra todos tus proyectos Django. GitHub Pages la publica gratis. Debes entregarla el 19 de noviembre de 2026 a las 5:00 p. m. El 1 de diciembre es el día de la graduación, no la fecha de entrega. Importante: Pages no ejecuta Django.

Proyecto final Fase 4 4 clases 8 pasos

Resultado final

  • Sitio estático con hero, sobre mí, skills, proyectos, certificaciones y contacto.
  • Cards para: To-Do, Notes, Calculadora, Mini CRM, Blog, HelpDesk y Dashboard.
  • Publicado en https://TU-USUARIO.github.io/TU-REPO/.
  • Rutas relativas correctas (sin / inicial).

Antes de empezar

  • Tienes cuenta de GitHub y Git instalado.
  • Completaste varios proyectos Django locales con capturas de pantalla.
  • Sabes que este tutorial es HTML/CSS/JS, no un nuevo proyecto Django.

Crítico: el portal enseña Django; Pages solo hospeda estáticos

No confundas los dos mundos

GitHub Pages sirve archivos estáticos: HTML, CSS, JS, imágenes. No instala Python. No ejecuta manage.py runserver. No monta SQLite. Tus apps Django viven en tu computadora (y en repos separados). El portafolio solo las documenta con capturas, textos y enlaces a GitHub.

Nota de Teacher Jim

El portal «Aprende Django con Teacher Jim» también es estático: te enseña a construir. Cuando alguien abre tu portafolio en Pages, ve tu escaparate profesional, no el servidor de desarrollo.

Crear la carpeta del portafolio

Qué vamos a hacer

Crear un repositorio/carpeta solo para el portafolio estático.

Estructura recomendada 
portfolio/
├── index.html
├── assets/
│   ├── css/
│   │   └── styles.css
│   ├── js/
│   │   └── main.js
│   └── img/
│       ├── hero.jpg          (opcional)
│       ├── todo.png
│       ├── notes.png
│       ├── calc.png
│       ├── crm.png
│       ├── blog.png
│       ├── helpdesk.png
│       └── dashboard.png
└── README.md

Por qué

Separar el portafolio de tus proyectos Django evita mezclar manage.py con un sitio que Pages sí puede servir.

JavaScript del menú móvil

portfolio/assets/js/main.js 
(function () {
  const toggle = document.querySelector(".nav-toggle");
  const nav = document.querySelector(".site-header nav");
  if (!toggle || !nav) return;
  toggle.addEventListener("click", function () {
    nav.classList.toggle("is-open");
  });
})();
6

Completar capturas y textos de cada card

Qué vamos a hacer

Para cada proyecto Django: captura PNG, descripción, tecnologías, enlace al repo (y video si tienes).

  • Usa la misma resolución aproximada (por ejemplo 1280×720).
  • No subas datos personales de clientes reales.
  • En la card menciona el problema que resuelve, no solo «app hecha en Django».

Nota de Teacher Jim

Si aún no tienes captura, deja un PNG de color sólido con el nombre del proyecto. Mejor una card completa que una imagen rota (ruta absoluta incorrecta).

7

Crear repo, commit y push

Terminal (dentro de portfolio/) 
git init
git branch -M main
git add .
git commit -m "Portafolio estático inicial en GitHub Pages"
# Crea el repo vacío en GitHub, luego:
git remote add origin https://github.com/TU-USUARIO/TU-REPO-PORTAFOLIO.git
git push -u origin main

Añade un .gitignore mínimo si trabajas con editores (.DS_Store, Thumbs.db). No subas secretos de Django aquí: este repo no debe contener settings.py con claves.

8

Activar GitHub Pages y verificar

Qué vamos a hacer

  1. En GitHub: Settings → Pages.
  2. Source: Deploy from a branch.
  3. Branch: main · folder: / (root).
  4. Save y espera 1–2 minutos.
  5. Abre https://TU-USUARIO.github.io/TU-REPO-PORTAFOLIO/.

Comprueba

El CSS carga, las imágenes se ven, el menú móvil funciona, y ningún enlace apunta a 127.0.0.1.

Error común

CSS 404 en Pages — Tenías /assets/... con barra inicial. Cámbialo a assets/... y vuelve a hacer push.

Errores comunes

Error común

Página en blanco / sin estilos — Rutas absolutas con / o archivos fuera de la carpeta publicada.

Error común

Intentar subir el proyecto Django a Pages esperando que funcione — No funcionará. Publica el portafolio estático; los repos Django quedan aparte.

Error común

Enlaces a localhost — Nunca pongas http://127.0.0.1:8000 como demo pública.
